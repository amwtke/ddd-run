## 6. 读模型(Read Models)

> 查询能力清单(对应 GET 端点 / 报表 / 列表页等)。
>
> **读模型不是聚合**,不受"一事务一聚合"约束。简单查询可复用 Repository,复杂查询可绕开聚合直接 SQL 或建独立 Read Model 表。
>
> 所有 `/ddd-storm` §6 的读模型必须被 `/ddd-model` 搬运到这里,否则查询侧会在后续实现阶段丢失。

| 查询(Query) | 输入 | 输出 DTO(字段) | 实现方式 | 归属上下文 |
|---|---|---|---|---|
| _待填充_ | | | | |

**实现方式枚举**:
- `单聚合直查` — 复用 `<Aggregate>Repository.findById`,映射聚合字段到 DTO(最简单)
- `跨聚合联合` — Application Service 拿多个聚合拼 DTO(业务维度跨聚合时用)
- `绕过聚合 SQL` — `JdbcTemplate` 直接查宽表,不经过聚合(列表 / 报表 / 高并发查询)
- `独立 Read Model 表` — 领域事件驱动构建的冗余表(真正 CQRS,读写模型分离)

**原则**:
1. 默认用"单聚合直查"。性能 / 需求驱动时升级
2. 读模型 DTO 属于 `interfaces.dto` 或 `application.query`,**不要**让聚合根直接序列化返回
3. 简单读模型**不需要** `/ddd-spec`,直接实现;复杂读模型(跨聚合 / 投影)可用 `/ddd-spec <查询名>` 生成查询 spec
